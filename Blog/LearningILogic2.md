1. Basic VB.NET Syntax
iLogic uses VB.NET, so you need to get comfortable with its style:

Lines don't need semicolons
Code blocks use keywords like If...End If, For...Next, Sub...End Sub
It reads almost like plain English, making it beginner-friendly

2. iLogic Parameters
This is the heart of iLogic. Parameters are the named dimensions and values in your Inventor model. You need to know how to:

Read a parameter: Dim width As Double = Parameter("Width")
Set a parameter: Parameter("Width") = 50
This is how your rules drive the model geometry

3. If/Else Logic for Configurations
Most iLogic rules are about switching configurations based on conditions:
```vb
vbIf Parameter("Type") = "Large" Then
    Parameter("Width") = 100
Else
    Parameter("Width") = 50
End If
```
This is probably the most used pattern in real iLogic work.
4. iProperties
iProperties are the document properties of your Inventor file (description, part number, material, etc.). Being able to read and write them is essential:
vbiProperties.Value("Project", "Description") = "Custom Bracket"
iProperties.Value("Project", "Part Number") = "BRK-001"
5. User Parameters vs. Model Parameters
Understanding the difference between:

User parameters — values you define manually to drive logic
Model parameters — dimensions generated by sketches/features
Knowing which to use and when keeps your rules clean and predictable.

6. String Handling
A lot of iLogic work involves building part numbers, descriptions, or file names dynamically:
```vb
vbDim partNum As String = "BRK-" & Parameter("Size") & "-" & Parameter("Material")
```
Concatenating and manipulating strings is used constantly.
7. Suppressing & Unsuppressing Features
One of the most powerful things iLogic can do is turn Inventor features on or off:
vbFeature.IsActive("Hole Pattern") = True
Feature.IsActive("Reinforcement Rib") = False
This lets you create highly configurable parts from a single model.
8. Component Activation in Assemblies
When working in assemblies, you'll want to control which components are active (visible/included):
```vb
vbComponent.IsActive("Lid:1") = True
Component.IsActive("Open Top:1") = False
```
9. MessageBox & InputBox for User Interaction
Simple ways to interact with users running your rules:
```vb
vbMsgBox("Model updated successfully!")
Dim userInput As String = InputBox("Enter the length:")
```
10. Rule Triggers & Organization
Understanding when rules run is critical:

Rules can trigger on parameter change, on document open, manually, etc.
Keeping rules short and focused on one job makes them much easier to maintain
Using external rules (shared across multiple files) is a huge time-saver in bigger projects


## Bonus Tips Specific to iLogic

The iLogic browser is your best friend — use it to organize and manage rules
Snippets panel inside the rule editor gives you pre-written code for common tasks — use it heavily when starting out
Test rules on simple models first before applying them to complex assemblies
Always save a backup before running a new rule on an important model


The good news is that iLogic is one of the more forgiving environments to learn programming in, because you get immediate visual feedback in your 3D model — you can see right away if your rule worked!
